## Project Setup Guide

This guide explains how to set up the LinkedIn Auto Job Applier locally (backend API, frontend dashboard, and legacy Flask UI).

---

## 1. Prerequisites

- **Python**: 3.10 or above (added to PATH)
- **Node.js & npm**: Recommended LTS version
- **MySQL** (or compatible) running locally
- **Google Chrome** installed in its default location

Optional but recommended:

- A virtual environment tool (`venv`, `conda`, etc.)

---

## 2. Clone the Repository

```bash
git clone <YOUR_REPO_URL>
cd Auto_job_applier_linkedIn
```

If you already have the repo, just `cd` into it.

---

## 3. Python Backend Setup

### 3.1 Create & Activate Virtual Environment (recommended)

```bash
python -m venv .venv
source .venv/bin/activate  # macOS / Linux
# .venv\Scripts\activate   # Windows (PowerShell / cmd)
```

### 3.2 Install Python Dependencies

From the repo root:

```bash
pip install -r requirements.txt
```

### 3.3 Configure Database

1. Make sure MySQL is running.
2. Create a database (example name from default config: `easy-apply`):

```sql
CREATE DATABASE `easy-apply` CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

3. (Optional) Set the `DATABASE_URL` environment variable instead of using the default:

```bash
export DATABASE_URL="mysql+pymysql://USER:PASSWORD@localhost:3306/easy-apply"
```

If `DATABASE_URL` is not set, the backend will use:

```text
mysql+pymysql://root:@localhost:3306/easy-apply
```

Adjust according to your local MySQL setup.

### 3.4 Backend Environment Variables

In the `backend` folder, copy the example env file and edit as needed:

```bash
cd backend
cp .env.example .env
```

Open `.env` and fill in the values (database, secrets, any API keys, etc.).

### 3.5 Run Database Migrations

The FastAPI app is configured to automatically apply Alembic migrations on startup. The first time you run the backend, it will create/update the schema.

You can also run Alembic manually if needed:

```bash
alembic upgrade head
```

from the `backend` directory (ensure `alembic.ini` is present and configured).

### 3.6 Start the FastAPI Backend

From the `backend` directory:

```bash
uvicorn main:app --reload
```

By default this serves at `http://127.0.0.1:8000`.

Health check:

```bash
curl http://127.0.0.1:8000/health
```

You should receive:

```json
{ "status": "ok" }
```

---

## 4. Frontend (React + Vite) Setup

### 4.1 Install Dependencies

From the `frontend` directory:

```bash
cd ../frontend
npm install
```

### 4.2 Configure Frontend Env

Copy the example `.env` file:

```bash
cp .env.example .env
```

Open `.env` and update the API base URL so it points to your backend (for example):

```env
VITE_API_URL=http://127.0.0.1:8000
```

### 4.3 Run Frontend Dev Server

```bash
npm run dev
```

By default, Vite runs at `http://localhost:5173`. The backend has CORS enabled for this origin.

## 5. Legacy Flask UI for Applied Jobs (Optional)

The original project includes a simple Flask app (`app.py`) that shows a table of applied jobs from CSV.

### 5.1 Ensure CSV Files Exist

`app.py` expects CSV files under:

```text
all excels/all_applied_applications_history.csv
```

Make sure this file and directory exist (they are typically generated by the bot).

### 5.2 Run the Flask App

From the repo root (and with your virtual environment active):

```bash
python app.py
```

Open your browser at:

```text
http://localhost:5000
```

## 6. Bot Configuration (High Level)

The bot behavior is configured through the `bot/config` files:

- `bot/config/personals.py`: Your personal details for applications.
- `bot/config/questions.py`: Answers to application questions and behavior flags.
- `bot/config/search.py`: Job search preferences and filters.
- `bot/config/settings.py`: Bot runtime behavior (stealth mode, intervals, background run, etc.).
- `bot/config/resume.py`: Resume paths and resume-related settings.

Before running the bot, open each of these files and adjust values to your needs.

---

## 7. Running the Bot

Once backend, database, and configuration are in place:

1. Ensure Chrome is installed and any required drivers / undetected-chromedriver dependencies are configured.
2. From the repo root (and virtual environment active), run:

```bash
python bot/runAiBot.py
```

The bot will open Chrome and begin applying to jobs based on your config.

### 7.1 Using Groq as the AI provider

This project can use Groq's OpenAI-compatible API so you can run models like Llama 3.3 70B:

- Get a Groq API key from their dashboard.
- In `backend/.env`, set:

```env
GROQ_API_KEY=your_groq_api_key_here
```

- In the web dashboard under **Settings → Secrets**, configure:
  - `ai_provider`: `groq`
  - `LLM API URL`: `https://api.groq.com/openai/v1`
  - `LLM model`: e.g. `llama-3.3-70b-versatile`
  - `LLM spec`: `openai`

The backend will inject `GROQ_API_KEY` into the bot's runtime config so the key is not stored in the database.

---

## 8. Common Issues & Tips

- **Database connection errors**: Check `DATABASE_URL` or MySQL credentials; ensure DB is running and the database exists.
- **CORS / API errors in frontend**: Verify `VITE_API_URL` in `frontend/.env` and that the FastAPI backend is running.
- **Missing CSV files** for the Flask UI: Run the bot first or create sample CSV files in `all excels/`.
- **Chrome / driver issues**: Ensure Chrome is up to date and that your undetected-chromedriver / chromedriver versions are compatible.
